# Omnave Design Language (ODL): Editorial AI Workspace

## Core Philosophy
Omnave is not a generic dashboard. It is an **Editorial AI Workspace**. The interface must feel calm, intelligent, focused, and intentional. Every screen is curated to guide the user toward meaningful learning without overwhelming them with raw data. 

**The ODL Promise:** Everything has a purpose.

## Editorial Composition
Every screen is composed like a modern editorial spread rather than a template of widgets.
*   **Visual Hierarchy:** Pages must possess one primary focal section, two supporting sections, and secondary sections. Important content demands more space.
*   **The "Why":** Forcing equal attention across every card creates cognitive overload. A clear hierarchy naturally guides the user's eyes from the most critical action down to supporting context.

## Visual Rhythm
The interface must create visual rhythm through intentional asymmetry while maintaining strict alignment.
*   **Avoid:** Equal-sized cards, uniform grids, repetitive layouts, and perfect symmetry.
*   **The "Why":** Symmetrical dashboards feel like admin panels. Asymmetry feels human-designed, premium, and prioritized.

## AI Philosophy
AI is a thoughtful learning companion, not a conversational chatbot living inside the interface.
*   **Contextual Intelligence:** AI surfaces insights quietly and contextually. It anticipates needs rather than waiting for prompts.
*   **The "Why":** Elevated intelligence should feel invisible. It should quietly improve the experience and empower the user without demanding attention or breaking the workspace immersion.

## The ODL Design Test
Before finalizing any screen or concept, it must pass these checks:
1. Does this screen feel like a curated editorial workspace or a generic admin dashboard?
2. Is there one undeniable focal point?
3. If the Omnave logo were removed, would this screen still feel uniquely like Omnave?
